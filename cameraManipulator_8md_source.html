4 The Radium Engine provides some classes that allow to extend the way one might interact with a camera in an
5 application.
6 
7 The class `Ra::Gui::CameraManipulator` defines the general interface for camera manipulator. All manipulators have
8 common properties that allow to switch from one manipulator to one other without disturbing the visual and interactive
9 behavior of the application.
10 
11 # Implementing a CameraManipulator
12 
13 In order to extend the set of `Ra::Gui::CameraManipulator` available in Radium or to develop a dedicated
14 `Ra::Gui::CameraManipulator` for a `Ra::Gui::Viewer`-based application, programmers are intended to do the following,
15 demonstrated by the class `FlightCameraManipulator`.
16 
17 1. Define the class that must inherits from `Ra::Gui::CameraManipulator` and, in order to receive interaction events,
18 from `Ra::Gui::KeyMappingManageable`. Note that a CameraManipulator is a `Q_OBJECT`
19 
20 ~~~{.cpp}
21 class RA_GUIBASE_API FlightCameraManipulator : public Ra::Gui::CameraManipulator,
22  public Ra::Gui::KeyMappingManageable<FlightCameraManipulator>
23 {
24  Q_OBJECT
25 ...
26 };
27 ~~~
28 
29 2. Implement the constructors (default, copy). Note that it is also very important to implement a constructor
30 that will take any `Ra::Gui::CameraManipulator` and will copy the base class before initializing the current
31 manipulator.
32 
33 ~~~{.cpp}
34 FlightCameraManipulator::FlightCameraManipulator( const Ra::Gui::CameraManipulator& other ) :
35  Ra::Gui::CameraManipulator( other ),
36  m_rotateAround( true ),
37  m_cameraRotateMode( false ),
38  m_cameraPanMode( false ),
39  m_cameraZoomMode( false ) {
40  m_flightSpeed = ( m_target - m_camera->getPosition() ).norm() / 10_ra;
41  initializeFixedUpVector();
42 }
43 
44 ~~~
45 
46 3. Implement the `Ra::Gui::KeyMappingManageable` part of the class. This implies defining the method
47 `void FlightCameraManipulator::configureKeyMapping_impl()` according to the semantic imposed
48 by `Ra::Gui::KeyMappingManageable`. It is recommended that, when implementing this keymapping initialisation callback,
49 a default configuration is defined and saved to the xml keymapping configuration file if this later does not already
50 contains a configuration. This will allow the users to edit and customize the proposed keymapping configuration.
51 
52 ~~~{.cpp}
53 #define KMA_VALUE( XX ) Gui::KeyMappingManager::KeyMappingAction Gui::FlightCameraManipulator::XX;
54 KeyMappingFlightManipulator
55 #undef KMA_VALUE
56 
57  void FlightCameraManipulator::configureKeyMapping_impl() {
58  m_keyMappingContext =
59  Ra::Gui::KeyMappingManager::getInstance()->getContext( "FlightManipulatorContext" );
60  if ( m_keyMappingContext.isInvalid() )
61  {
62  // The context is undefined, add a default configuration to the keymapping stystem
63  Ra::Gui::KeyMappingManager::getInstance()->addAction(
64  "FlightManipulatorContext", "", "", "LeftButton", "", "FLIGHTMODECAMERA_ROTATE" );
65  Ra::Gui::KeyMappingManager::getInstance()->addAction( "FlightManipulatorContext",
66  "",
67  "ShiftModifier",
68  "LeftButton",
69  "",
70  "FLIGHTMODECAMERA_PAN" );
71  Ra::Gui::KeyMappingManager::getInstance()->addAction( "FlightManipulatorContext",
72  "",
73  "ControlModifier",
74  "LeftButton",
75  "",
76  "FLIGHTMODECAMERA_ZOOM" );
77  Ra::Gui::KeyMappingManager::getInstance()->addAction(
78  "FlightManipulatorContext", "Key_A", "", "", "", "FLIGHTMODECAMERA_ROTATE_AROUND" );
79  m_keyMappingContext =
80  Ra::Gui::KeyMappingManager::getInstance()->getContext( "FlightManipulatorContext" );
81  }
82 
83 #define KMA_VALUE( XX ) \
84  XX = Ra::Gui::KeyMappingManager::getInstance()->getActionIndex( m_keyMappingContext, #XX );
85  KeyMappingFlightManipulator
86 #undef KMA_VALUE
87 }
88 ~~~

92 # Using a CameraManipulator
93 Using a `Ra::Gui::CameraManipulator` in a `Ra::Gui::Viewer`-based application is quite straightforward.
94 
95 If one wants to set a first camera manipulator to a viewer
96 ~~~{.cpp}
97 myViewer->setCameraManipulator(
98  new FlightCameraManipulator( width, height );
99 ~~~
100 
101 If one wants to change the manipulator while keeping the actual visual state
102 ~~~{.cpp}
103 myViewer->setCameraManipulator(
104  new FlightCameraManipulator( *( m_viewer->getCameraManipulator() ) ) );
105 ~~~
