4 # Installation directory
5 By default, `${CMAKE_INSTALL_PREFIX}` is set as follow:
6 
7 - For release build :
8  ~~~{.cmake}
9  set(RADIUM_BUNDLE_DIRECTORY ${CMAKE_CURRENT_SOURCE_DIR}/Bundle-${CMAKE_CXX_COMPILER_ID})
10  ~~~
11 - For Debug orRelWithDebInfo build
12  ~~~{.cmake}
13  set(RADIUM_BUNDLE_DIRECTORY ${CMAKE_CURRENT_SOURCE_DIR}/Bundle-${CMAKE_CXX_COMPILER_ID}-${CMAKE_BUILD_TYPE})
14  ~~~

16 # Radium Libraries
17 ## Overview
18 Radium is split in 5 libraries: Core, Engine, GuiBase, UI and PluginBase.
19  The compilation of each library is controlled by the following cmake options
20 ~~~{.cmake}
21 option(RADIUM_GENERATE_LIB_CORE "Include Radium::Core in CMake project" ON)
22 option(RADIUM_GENERATE_LIB_IO "Include Radium::IO in CMake project" ON)
23 option(RADIUM_GENERATE_LIB_ENGINE "Include Radium::Engine in CMake project" ON)
24 option(RADIUM_GENERATE_LIB_GUIBASE "Include Radium::GuiBase in CMake project" ON)
25 option(RADIUM_GENERATE_LIB_PLUGINBASE "Include Radium::PluginBase in CMake project" ON)
26 ~~~
27 ## Dependencies between libraries
28 ~~~{.cmake}
29 add_dependencies (${ra_engine_target} PUBLIC Radium::Core)
30 add_dependencies (${ra_io_target} PUBLIC Radium::Core)
31 add_dependencies (${ra_pluginbase_target} Radium::Core Radium::Engine)
32 add_dependencies (${ra_guibase_target} PUBLIC Radium::Core Radium::Engine Radium::PluginBase Radium::IO)
33 ~~~
34 \warning Consistency of `RADIUM_GENERATE_LIB_***` options is not checked wrt. the dependencies.
35 
36  - When enabled using `RADIUM_GENERATE_LIB_***`, each library has a compilation target: `Core`,
37  `Engine`, ...
38

81 Whats is happening here, is that the dependencies of Core are fetched using git, then
82 configured using cmake, built, and installed to ${CMAKE_INSTALL_PREFIX} of Radium.
83 \see File `cmake/externalFunc.cmake` for technical details.
84 
85 In order to save compilation time, dependencies are processed at the first run, and then skipped.
86 To force processing again the dependencies of a given library, turn on the associated option
87 ~~~
88 OPTION( RADIUM_SKIP_${NAME_UPPER}_EXTERNAL "[addExternalFolder] Skip updating ${NAME}::external (disable for rebuild)" ON)
89 ~~~
90 where `$NAME_UPPER` is `CORE`, `ENGINE`, `GUIBASE`, `PLUGINBASE` or `IO`.
91 \note The option `RADIUM_SKIP_${NAME_UPPER}_EXTERNAL` is generated only if `RADIUM_GENERATE_LIB_${NAME_UPPER}` is `ON`.

96 To this end, just provide the corresponding '*_DIR' to cmake at configuration time (with '-D' option, configuration file or toolchain file, please refer to cmake documentation).

136 ## Building and installing Radium dependencies once for all
137 Radium dependencies can be built and installed alone, without building the whole Radium project.
138 This will allow to use them as user provided external dependencies for Radium but also to develop other software that will use the same dependencies as Radium.
139 
140 The file `external/CMakeLists.txt` could be used as a standalone project to install all the Radium dependencies in any location by doing the following :
141 ~~~{.bash}
142 mkdir BuildRadiumDependencies && cd BuildRadiumDependencies
143 cmake -DCMAKE_INSTALL_PREFIX=/path/to/install /path/to/Radium/external
144 ~~~
145 
146 If not given on the command line, the installation directory is set by default to, `{CMAKE_CURRENT_BINARY_DIR}/Bundle-${CMAKE_CXX_COMPILER_ID}-${CMAKE_BUILD_TYPE}`.
147 
148 Compiling Radium using the pre-installed dependencies should be made as described in the section above.
149 
150 \warning You have to take care of the consistency of the external dependencies, e.g. it's not possible to use your version of globjects without providing your version of eigen, otherwise you will have mixed version in Radium.
