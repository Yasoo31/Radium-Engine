5 # Usage
6 
7 You first need to add into every configuration file (in `Configs/` folder) an entry which will bind your action to a key.
8 This file as an only `keymaps` node, with `keymap` per binding.
9 
10 ~~~{.xml}
11 <keymaps>
12  <keymap context="CONTEXT" key="KEY" modifiers="MODIFIERS" buttons="BUTTONS" wheel="WHEEL" action="ACTION" />
13  <!-- as many entry as needed -->
14 </keymaps>
15 ~~~
16 
17 Default values are in \[ \]
18 
19 * `ACTION` *mandatory* represents the KeyMappingAction enum's value you want to trigger.
20 * `CONTEXT` \[AppContext\] epresents the context, for instance the class that will try to catch this keymapping
21 * `KEY` \[no def\] represents the key that need to be pressed to trigger the event (ie `Key_Z`, for example), "" or "-1" or absent correspond to no key needed.
22 * `MODIFIERS` \[NoModifiers\]represents the modifier used along with key or mouse button (needs to be a `Qt::Modifier` enum value) to trigger the action.
23 Multiples modifiers can be specified, separated by commas as in `"ControlModifier,ShiftModifier"`.
24 * `BUTTONS` \[NoButtons\] represents the button to trigger the event (i.e. `LeftButton`, for example).
25 * `WHEEL` \[false\] if true, it's a wheel event ! (anything else is false).
26 
27 If only a key is defined then it's a `keyPressedEvent`.
28 If buttons is defined, then it's a `mouse[move/press/release]Event`, that optionally take modifiers and key into account.
29 If wheel is true, then it's a wheel event, that optionally take modifiers, buttons and key into account.
30 
31 On the implementation side, your class `C` need (could) derive from `KeyMappingManageable<C>`, it defines the static member variable
32 `m_keyMappingContext`, and the static function `configureKeyMapping()`, that calls `configureKeyMapping_impl()`, which have to be implemented in your class `C`.
33 
34 Then you need to define your specific actions as static member of your class
35 
36 ~~~{.cpp}
37 class MyClass :public KeyManageable<MyClass> {
38  friend class KeyMappingManageable<MyClass>;
39 
40 // callback when a configuration file is loaded, could check if the context and action are present.
41 static void configureKeyMapping(){
42  m_keyMappingContext = KeyMappingManager::getInstance()->getContext( "MyClassContext" );
43  m_myAction = KeyMappingManager::getInstance()->getActionIndex( "MyActionName" );
44 }
45 
46 //[...]
47 static KeyMappingManager::KeyMappingAction m_myAction;
48 // [...]
49 };
50 
51 
52 // typically in .ccp file
53 KeyMappingManager::KeyMappingAction m_myAction;
54 
55 // then typically in main baseApplication ctor or Viewer ctor, but after KeyMappingManager instance is created :
56 KeyMappingManager::getInstance()->addListener(MyClass:configureKeyMapping);
57 
58 ~~~
59 
60 It could be done with a specific macro :
61 
62 ~~~{.cpp}
63 // in header
64 #define KeyMappingMyClass \
65  KMA_VALUE( MY_ACTION )
66 
67 #define KMA_VALUE( XX ) static KeyMappingManager::KeyMappingAction XX;
68  KeyMappingMyClass
69 #undef KMA_VALUE
70 
71 // in source
72 #define KMA_VALUE( XX ) Gui::KeyMappingManager::KeyMappingAction MyClass::XX;
73 KeyMappingMyClass
74 #undef KMA_VALUE
75 
76 void MyClass::configureKeyMapping() {
77  m_keyMappingContext = Gui::KeyMappingManager::getInstance()->getContext( "MyClassContext" );
78  if ( m_keyMappingContext.isInvalid() )
79  LOG( logINFO ) << "MyClassContext not defined (maybe the configuration file does not contain it";
80 #define KMA_VALUE( XX ) \
81  XX = Gui::KeyMappingManager::getInstance()->getActionIndex( m_keyMappingContext, #XX );
82  KeyMappingMyClass
83 #undef KMA_VALUE
84 }
85 
86 ~~~

88 # Implementation note
89 
90 The viewer is the main entry point to dispatch key and mouse event.
91 The idea is that at a key press or mouse press event, the viewer is capable of determining which class will receive the events.
92 
93 For instance see `Viewer.cpp` `Viewer::mousePressEvent`
94 
95 ~~~{.cpp}
96 //[...]
97  auto buttons = event->buttons();
98  auto modifiers = event->modifiers();
99  auto key = activeKey();

101  // nothing under mouse ? juste move the camera ...

103  {
104  auto accepted = m_camera->handleMousePressEvent( event, buttons, modifiers, key );
105  if ( accepted ) { m_activeContext = m_camera->getContext(); }
106  else
107  //[...]

110 # Key mapping and inheritence.
111 If you want to define a derived class that inherits a base class with key mapping, and you want to have key mapping management in this derived, you have to consider the following implementation details:
112 
113 * Base and derived classes will have two different contexts.
114 * Derived `setupKeymappingcallbacks` should call base `setupKeymappingcallbacks`, hence derived `configureKeymapping_impl` should only care about its own `KeyMappingAction`.
115 
116 Here is an example snippets.
117 
118 ~~~{.cpp}
119 class MyViewer : public Ra::Gui::Viewer, public Ra::Gui::KeyMappingManageable<MyViewer>
120 {
121  using baseKeyMapping = Ra::Gui::KeyMappingManageable<Ra::Gui::Viewer>;
122  using thisKeyMapping = Ra::Gui::KeyMappingManageable<MyViewer>;
123  void setupKeyMappingCallbacks() override;
124  friend class Ra::Gui::KeyMappingManageable<MyViewer>;
125 
126 //[...]
127 #define KeyMappingMyViewer \
128  KMA_VALUE( MY_ACTION )
129 //[...]
130 }
131 
132 void MyViewer::setupKeyMappingCallbacks() {
133  // Setup keymapping for base
134  base::setupKeyMappingCallbacks();
135 
136  // then extend
137  auto keyMappingManager = Gui::KeyMappingManager::getInstance();
138  keyMappingManager->addListener( thisKeyMapping::configureKeyMapping );
139 }
140 
141 void MyViewer::configureKeyMapping_impl() {
142  auto keyMappingManager = Gui::KeyMappingManager::getInstance();
143  thisKeyMapping::m_keyMappingContext = keyMappingManager->getContext( "MyViewerContext" );
144  if ( thisKeyMapping::m_keyMappingContext.isInvalid() )
145  {
146  LOG( logINFO ) << "MyViewerContext not defined (maybe the configuration file do not contains it)";
147  return;
148  }
149 
150 #define KMA_VALUE( XX ) XX = keyMappingManager->getActionIndex( thisKeyMapping::m_keyMappingContext, #XX );
151 
152  KeyMappingMyViewer
153 
154 #undef KMA_VALUE
155 }
156 ~~~
